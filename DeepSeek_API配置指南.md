# DeepSeek API 配置示例

## 基本配置

### 1. API 基础设置
- **API Base URL**: `https://api.deepseek.com/v1`
- **模型**: `deepseek-chat`
- **API密钥**: 从 https://platform.deepseek.com/apiKeys 获取

### 2. 推荐参数设置
- **温度参数**: 0.3 (翻译任务推荐较低温度，确保一致性)
- **翻译风格**: 根据需求选择
- **源语言**: 自动检测或指定
- **目标语言**: 中文或其他目标语言

## 使用示例

### 示例1：英文翻译成中文（标准风格）
```
API Base URL: https://api.deepseek.com/v1
API Key: [您的DeepSeek API密钥]
模型: deepseek-chat
源文本: Hello, how are you today? I hope everything is going well.
源语言: 英文
目标语言: 中文
翻译风格: 标准
温度参数: 0.3
```

**预期输出**: "你好，你今天怎么样？我希望一切都顺利。"

### 示例2：中文翻译成英文（正式风格）
```
API Base URL: https://api.deepseek.com/v1
API Key: [您的DeepSeek API密钥]
模型: deepseek-chat

翻译风格: 正式
温度参数: 0.3
```

**预期输出**: "Artificial intelligence technology is developing rapidly, bringing new opportunities to various industries."

### 示例3：日文翻译成中文（口语化风格）
```

API Key: [您的DeepSeek API密钥]
模型: deepseek-chat
```

### 批量翻译设置
对于长文本或批量翻译，建议：
- 将长文本分段处理
- 使用较低的温度参数（0.1-0.3）
- 选择合适的翻译风格保持一致性

## 性能优化

### 1. 温度参数选择
- **0.1-0.3**: 最稳定的翻译，适合正式文档
- **0.4-0.6**: 平衡的翻译，适合一般用途
- **0.7-1.0**: 更有创造性的翻译，适合文学内容

### 2. 翻译风格建议
- **标准**: 日常对话、普通文本
- **正式**: 商务文档、学术论文
- **口语化**: 社交媒体、日常交流
- **学术**: 研究论文、技术文档
- **商务**: 商业邮件、合同文件

### 3. 语言检测
- 使用"自动检测"可以简化操作
- 对于混合语言文本，建议手动指定源语言

## 费用控制

DeepSeek API的费用相对较低，但建议：
- 先进行小量测试
- 监控API使用量
- 合理设置最大token数
- 批量处理时适当分段

## 故障排除

### 常见问题

1. **API连接失败**
   - 检查网络连接
   - 验证API密钥有效性
   - 确认API Base URL正确

2. **翻译质量不佳**
   - 调整温度参数
   - 选择合适的翻译风格
   - 检查源文本清晰度

3. **响应速度慢**
   - 减少单次文本长度
   - 检查网络延迟
   - 考虑使用更快的模型

## 获取API密钥

1. 访问 https://platform.deepseek.com
2. 注册/登录账户
3. 进入API Keys管理页面
4. 创建新的API密钥
5. 安全保存密钥（不会再次显示）
